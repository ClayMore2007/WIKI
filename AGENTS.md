# ClayMore 知识库 LLM 维护规则

这是低敏个人知识库。维护目标是：有用、有来源、可链接、易更新。

## 硬边界

- 不存密码、2FA 密钥、API Key、助记词、私钥、完整银行卡信息。
- 高敏身份信息不要写在这里，放到 `ClayMore-Private-Wiki`。
- 股票、财务、健康、账号、身份、法律相关内容视为高风险，必须标注来源和不确定性。
- 不擅自删除、合并、改主标题、统一别名或移动外部同步目录。

## 默认读取

- 日常操作先读 [[00-meta-规则/00-快速入口]]。
- 整体原则见 [[00-meta-规则/10-知识库原则]]。
- 新增、修改或删除规则前，先读 [[00-meta-规则/90-规则维护规则]]。

## 默认流程

- 用户说“整理资料”“整理 wiki”“整理我的 wiki”或类似指令时，按 [[00-meta-规则/00-快速入口]] 执行。
- 用户提到“更新股票”“更新我的股票”“同步同花顺”或类似泛指股票更新的指令时，优先在 Obsidia 根目录运行 `python .\Tools-工具\update_ths_stocks.py`，同时更新自选股和我的持仓；规则见 [[00-meta-规则/同花顺/23-同花顺自选股更新流程]]。
- 用户要求“表格整理股票信息”“输出股票 Markdown 汇总”或类似指令时，按同花顺股票更新规则更新 `10-股票投资/02-股票研究流/我的股票信息.md`，只汇总脚本目标页已有字段。
- 用户明确只提到“更新自选股”“同花顺自选股”时，运行 `python .\Tools-工具\update_ths_stocks.py --only self`。
- 用户明确只提到“更新持仓”“我的持仓”“同花顺持仓板块”时，运行 `python .\Tools-工具\update_ths_stocks.py --only holdings`。
- 用户投放的新资料优先从 `80-raw-原始资料/待整理/` 读取和整理。
- 外部同步目录如 `Cubox/` 保持原貌，不直接移动、重命名或删除；按 [[00-meta-规则/21-Cubox接入流程]] 生成 raw 副本和 wiki 沉淀。
- raw 来源笔记放入 `80-raw-原始资料/`。
- assets 附件放入 `90-assets-附件/`。
- 过期或停用页面放入 `99-archive-归档/`。

## 写作边界

- 优先更新已有页面，再创建新页面。
- 主知识页面只放稳定知识内容；整理过程进入 `00-log-整理日志/`。
- 分类树由用户控制；遇到没有对应目录或现有分类无法覆盖的资料时，先写入整理报告并询问用户。
- 来源冲突时保留冲突，不要擅自合并成单一结论。
- 清单型资料必须检查完全重复、别名重复、错别字重复和系列/续作关系；合并前必须询问用户。
