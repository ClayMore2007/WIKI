# Cubox 接入流程

`Cubox/` 是外部软件同步目录，作为 external raw source 处理。

## 触发词和边界

| 用户说法                        | 默认动作                                                      |
| --------------------------- | --------------------------------------------------------- |
| “整理 Cubox”“整理资料”“整理最新资料”    | 触发完整流程：先从 `Cubox/` 提取到 raw，再基于 raw 分析并沉淀到 wiki 或索引        |


## 硬规则

- 不直接移动、重命名或删除 `Cubox/` 中的文件。
- 从 `Cubox/` 生成的 raw 副本、metadata、转写稿和提取模板都必须放在 `80-raw-原始资料/Cubox/` 下对应分类。
- 先解析 frontmatter `tags`，再判断分类。
- 可映射标签优先；标题、URL、正文关键词只能兜底。
- 标签与标题/内容冲突时，按标签归档，并在报告中标记冲突。
- 标签无法映射时，不自行创建新目录；写入报告并询问用户。
- 网页和视频资料需要按 [[00-meta-规则/22-内容提取流程]] 标记内容质量。

## 敏感性

| 类型 | 处理方式 |
|---|---|
| 低敏公开资料 | 可以进入主库 |
| 股票、财务、健康、法律资料 | 可以索引和摘要，必须标注来源和不确定性 |
| 保单、证件、账号、身份资料 | 只做低敏索引，不写密码、密钥、完整证件号 |
| 无法判断敏感性 | 跳过，写入整理报告 |

## raw 副本

raw 副本保留：

- Cubox 原路径。
- 原文件名。
- 来源 URL。
- 收藏日期或文件日期。
- 原始 tags。
- 摘要、正文摘录或内容质量标记。

## 整理报告

每批 Cubox 报告必须包含：

- 本次范围。
- 分类审计表。
- 处理追踪表：Cubox 原文件 → raw 副本 → wiki 页面 → 状态。提取模式下 wiki 页面可留空，并标记“已复制 raw”或“已提取 raw”。
- 处理结果。
- 风险提示。
- 待确认事项。

状态字段固定为：未处理 / 已复制 raw / 已提取 raw / 已沉淀 wiki / 已跳过。

## 股票资料关系图

处理 `Cubox/` 股票资料并生成关系图时：

- 默认使用 Obsidian Canvas，关系图放在对应详细报告同目录，命名为 `原报告标题-关系图.canvas`；已有更具体名称的 Canvas 可以沿用原名。
- 关系图必须基于该资料正文单独生成，禁止套用统一占位模板；节点只整理产业链环节、公司/标的、工具模块、业务链条和关键关系。
- Canvas 节点应短句化，优先写“环节 + 公司/标的 + 关系/待核验点”；不得写大段研究说明，也不得只写“资料摘要、关系图主线、内容节点”等通用占位文字。
- Canvas 文件必须是合法 JSON，包含 `nodes` 和 `edges`；节点优先使用 text/file 类型，file 节点链接回来源详细报告。
- 生成后必须解析 Canvas 结构校验：节点数不能为 0；边引用的 `fromNode`/`toNode` 必须都存在；`x/y/width/height` 必须是 number，画布边界不得出现数千以上的异常坐标。
- Excalidraw 不再作为默认格式；仅在用户明确要求手绘展示图时使用。
- 关系图必须标注来源详细报告、低置信/待核验口径和“不构成投资建议”风险。
- 股票视频字幕、OCR 或资料正文中抽取出的公司名，若存在识别不确定、同音词、错别字或简称混淆，优先用本地持仓/自选匹配，再用 `10-股票投资/02-股票研究流/基础数据/A股公司名称库.csv` 校验证券简称、代码和市场；名称库可通过 `python .\Tools-工具\update_a_stock_name_library.py` 刷新。
- 本地名称库、东方财富行情接口或交易所公开列表只作为公司名/证券简称校验层，不用于确认供应链关系、客户、订单、份额、业绩、机构持仓或投资结论；校验后仍无法确认的名称必须标记“疑似”或“需核验”。
- 生成或更新关系图后，必须同步更新 `10-股票投资/01-Cubox资料流/股票资料汇总.md` 的 `关系图` 列。
- 暂无关系图的资料在汇总页中用 `-` 占位，不要链接不存在的页面。

更新 `10-股票投资/01-Cubox资料流/股票资料供应链关系总览.canvas` 时：

- 总览 Canvas 主体继续按“主题/产业链 → 环节 → 公司/标的/模块”组织，只保留公司、产业链和关键供应链关系，不写大段研究说明，也不生成 Markdown 关系表。
- 画布右侧必须增加“涉及公司 × 我的持仓/自选”旁栏，数据源使用 `10-股票投资/02-股票研究流/我的持仓.md` 和 `10-股票投资/02-股票研究流/我的自选.md`。
- 旁栏固定分组为：`当前持仓命中`、`自选命中（不含持仓）`、`资料涉及但未在持仓/自选池（待核验）`。
- 每条命中记录优先写“公司名（代码）：来源主题/产业链环节”，保留“疑似”标记；未命中组只作为待核验线索，不作为推荐清单。
- 旁栏总览节点必须写明涉及公司/标的数量、持仓命中数量、自选非持仓命中数量、未命中数量、数据源和“不构成投资建议”口径。
