# Cubox 接入流程

`Cubox/` 是外部软件同步目录，作为 external raw source 处理。

## 触发词和边界

| 用户说法 | 默认动作 |
|---|---|
| “提取 Cubox”“提取资料”“只提取 Cubox” | 只从 `Cubox/` 生成或更新 raw 副本，补充正文、metadata、转写和内容质量标记；不更新 wiki |
| “分析已提取资料”“整理已提取资料”“基于 raw 分析” | 不扫描 `Cubox/`，只读取 `80-raw-原始资料/Cubox/` 下已有资料并整理到 wiki |
| “整理 Cubox”“整理资料”“整理最新资料” | 触发完整流程：先从 `Cubox/` 提取到 raw，再基于 raw 分析并沉淀到 wiki 或索引 |

“提取”与“分析”是互斥阶段；只有“整理资料 / 整理 Cubox”默认包含两个阶段。

## 硬规则

- 不直接移动、重命名或删除 `Cubox/` 中的文件。
- 从 `Cubox/` 生成的 raw 副本、metadata、转写稿和提取模板都必须放在 `80-raw-原始资料/Cubox/` 下对应分类；不得落入 `80-raw-原始资料/股票/`、`80-raw-原始资料/技术/` 等通用 raw 目录。
- 先解析 frontmatter `tags`，再判断分类。
- 可映射标签优先；标题、URL、正文关键词只能兜底。
- 标签与标题/内容冲突时，按标签归档，并在报告中标记冲突。
- 标签无法映射时，不自行创建新目录；写入报告并询问用户。
- 网页和视频资料需要按 [[00-meta-规则/22-内容提取流程]] 标记内容质量。

## 标签映射

| Cubox 标签 | raw 去向 | wiki 去向 |
|---|---|---|
| 股票 | `80-raw-原始资料/Cubox/股票/` | `10-股票投资/` |
| 旅游 / 成都 / 露营 / 川西 / 龙泉山 | `80-raw-原始资料/Cubox/旅游/` | `20-旅游出行/` |
| AI / ClaudeCode | `80-raw-原始资料/Cubox/技术/AI/` | `30-技术知识/AI与大模型/` |
| 技术 / 编程 / 工具 / cubox | `80-raw-原始资料/Cubox/技术/` | `30-技术知识/` |
| 动画 / 电影 / 游戏 | `80-raw-原始资料/Cubox/动画电影/` | `40-动画电影/` |
| 中医 / 道家 / 健康 / 医疗 / 黄精 | `80-raw-原始资料/Cubox/健康养生/` | `60-健康养生/` |
| 摄影 | `80-raw-原始资料/Cubox/兴趣创作/摄影/` | `70-兴趣创作/摄影/` |
| 保险 | `80-raw-原始资料/Cubox/个人信息/保险/` | `50-个人低敏信息/保险/` |
| 字母圈 | `80-raw-原始资料/Cubox/未分类/` | `98-未分类/` |

新增标签映射前，先确认分类是否长期有用、是否能并入现有分类。

## 敏感性

| 类型 | 处理方式 |
|---|---|
| 低敏公开资料 | 可以进入主库 |
| 股票、财务、健康、法律资料 | 可以索引和摘要，必须标注来源和不确定性 |
| 保单、证件、账号、身份资料 | 只做低敏索引，不写密码、密钥、完整证件号 |
| 无法判断敏感性 | 跳过，写入整理报告 |

## raw 副本

raw 副本保留：

- Cubox 原路径。
- 原文件名。
- 来源 URL。
- 收藏日期或文件日期。
- 原始 tags。
- 摘要、正文摘录或内容质量标记。

## 整理报告

每批 Cubox 报告必须包含：

- 本次范围。
- 分类审计表。
- 处理追踪表：Cubox 原文件 → raw 副本 → wiki 页面 → 状态。提取模式下 wiki 页面可留空，并标记“已复制 raw”或“已提取 raw”。
- 处理结果。
- 风险提示。
- 待确认事项。

状态字段固定为：未处理 / 已复制 raw / 已提取 raw / 已沉淀 wiki / 已跳过。
