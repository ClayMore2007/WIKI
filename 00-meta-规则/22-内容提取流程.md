# 内容提取流程

网页、视频、文章进入长期知识层前，应尽量提取实际内容。默认策略是：能获取正文就获取正文；公开视频能下载就下载，能转写就转写；只有在访问、权限、工具、依赖或时间成本受阻时，才降级为 metadata、partial 或 title-only。只有标题的资料可以进入 raw 或索引，但不应直接写成长期结论。

视频文件只作为提取字幕或机器转写的临时中间产物；raw 层长期保留 metadata、Markdown 字幕/转写稿、分段 JSON 和必要的来源记录，不保留 mp4/m4a。字幕或转写提取完成后，将转写稿转成 `_transcript.md` 作为 Obsidian 可读 raw 原始文件，再删除临时视频/音频文件；若转写失败，也删除不完整媒体文件，并在报告中记录失败原因和待补提取状态。

## 内容质量

| 标记 | 含义 |
|---|---|
| `content: full` | 获取完整正文、字幕或机器转写稿 |
| `content: metadata` | 获取标题、简介、作者、发布时间、标签、互动数据等 metadata，但没有正文或字幕 |
| `content: partial` | 获取部分正文、简介或字幕 |
| `content: title-only` | 只有标题和标签 |
| `content: none` | 无实质内容，例如直播入口 |

## 提取顺序

1. 先读来源文件已有内容。
2. 再尝试获取原网页、原文、视频简介或平台 metadata。
3. 对公开视频资料，默认继续尝试下载媒体并转写，除非来源不可访问、需要登录/验证码/私密权限、脚本失败、依赖缺失且不适合安装，或用户明确要求只做轻量索引。
4. 下载或转写失败时，记录失败原因和已获得的最高内容质量，不强行生成结论。
5. 仍无法提取时，标记内容质量，不强行生成结论。

## 默认深度

| 场景 | 默认动作 |
|---|---|
| 用户说“提取资料”“只提取”“提取 Cubox” | 对网页尽量抓正文；对视频默认下载并转写；产物只进入 raw，不沉淀 wiki |
| 用户说“分析资料”“分析已提取资料”“整理已提取资料” | 只读取已提取的 raw 内容做分析整理；不重新扫描 Cubox 或外部来源 |
| 用户说“整理资料”“整理最新资料”“整理这条资料” | 执行完整流程：先提取到 raw，再基于 raw 分析，符合条件的内容沉淀到 wiki 或索引 |
| 用户说“先入库”“只建索引”“快速扫一遍” | 可只做 raw、metadata 和索引，不进入完整分析 |
| 涉及股票、财务、健康、法律 | 仍默认提取正文/转写，但输出必须区分“来源声称 / 已核验事实 / 待核验” |
| 下载或转写耗时过长 | 可先保存 metadata，并在报告中写明“待下载/待转写”及原因 |

## 分来源规则

- 抖音：先用 `extract_douyin.py` 获取 metadata（标题、作者、简介、发布时间、时长、话题标签、互动数据、播放地址）；默认继续执行 `--download --transcribe` 下载并转写。分享页打不开时尝试 `/share/video/<id>` 加移动端 UA；如果没有有效播放地址或转写依赖不可用，记录原因并降级。
- B 站：先用 `yt-dlp --dump-json <url>` 获取完整 metadata，包括作者、发布时间、简介、标签、时长、互动数据和字幕列表；默认优先获取字幕，字幕为空时继续尝试下载音频并转写。失败时标记为 `content: metadata` 或 `content: partial`，并写入失败原因。
- 网页文章：获取原文，去掉导航、广告、评论。
- 小红书：优先使用 Cubox 已保存正文；链接失效时标记。

## B 站提取判定

| 可获得内容 | 标记 | 处理方式 |
|---|---|---|
| 完整字幕或转写稿 | `content: full` | 可以提炼长期知识，但 ASR 专有名词仍需核验 |
| 完整简介 + 标签 + 互动数据 | `content: metadata` | 可以更新 raw、索引和主题线索；不写确定结论 |
| Cubox 卡片摘要 | `content: partial` | 只能作为线索 |
| 只有标题 | `content: title-only` | 只进 raw 或索引 |

如果 B 站视频没有字幕，但内容重要，整理报告必须记录”需音频转写”，不要把简介当作完整视频内容。

## 抖音提取判定

| 可获得内容 | 标记 | 处理方式 |
|---|---|---|
| 完整转写稿 | `content: full` | 可以提炼长期知识，但 ASR 专有名词（公司名、产品名、股票名）仍需核验 |
| 完整 metadata + 简介 + 话题标签 + 互动数据 | `content: metadata` | 可以更新 raw、索引和主题线索；不写确定结论 |
| 分享文案 + 标题 | `content: partial` | 只能作为线索 |
| 只有标题和 URL（页面打不开或反爬） | `content: title-only` | 只进 raw 或索引 |

如果抖音视频没有有效播放地址，或者分享页返回 `encrypt_data_miss`，不尝试破解签名；改用 Cubox 已保存的分享文案，标记为 `content: partial`。

可复用脚本：

抖音：

```powershell
# 只提取 metadata
python "Tools-工具/video/抖音视频内容提取/scripts/extract_douyin.py" "<抖音链接>" --out-dir .

# 下载 + 转写
python "Tools-工具/video/抖音视频内容提取/scripts/extract_douyin.py" "<抖音链接>" --out-dir . --download --transcribe --model base
```

B 站：

```powershell
# 只提取 metadata
python "Tools-工具/video/B站视频内容提取/scripts/extract_bilibili.py" "<B站链接>" --out-dir .

# 下载音频 + 转写
python "Tools-工具/video/B站视频内容提取/scripts/extract_bilibili.py" "<B站链接>" --out-dir . --download-audio --transcribe --model base
```

脚本输出：

- `<id>_metadata.json`（两平台通用）
- `<id>.mp4`（抖音）或 `<id>.m4a`（B 站）：只作为临时中间文件，字幕/转写完成后删除
- `<id>_transcript.md`（带时间戳的原始转写稿，raw 层长期保留格式）
- `<id>_segments.json`（结构化分段）
- `<id>_notes_template.md`（整理稿模板）

## 禁止事项

- 不用标题替代正文。
- 不把 `title-only` 资料写成确定事实。
- 不把视频 metadata（B 站或抖音）当作完整视频观点。
- 不为 `content: none` 的资料创建独立 wiki 结论页。
