## 默认流程

1. 先扫描 `80-raw-原始资料/待整理/`。
2. 再对外部同步目录做增量检查，例如 `Cubox/`。
3. 外部同步目录保持原貌，不移动、不重命名、不删除。
4. 先保留 raw/source，再沉淀 wiki 页面。
5. 过程、异常、候选合并、用户确认事项写入 `00-log-整理日志/`。
6. 新整理产物必须有整理日期；来源发布日期、收藏日期或文件日期能确认就记录，补不上时写“来源日期未知”，不为补日期卡住整理。
7. 主知识页只放稳定知识、索引、判断、待核验问题和行动。

## 快捷脚本

- 股票更新优先使用 Obsidia 工程下的脚本：`python .\Tools-工具\update_ths_stocks.py`。
- 泛指“更新股票 / 更新我的股票 / 同步同花顺”时，默认同时更新自选股和我的持仓。
- 明确只更新自选股时使用 `python .\Tools-工具\update_ths_stocks.py --only self`。
- 明确只更新持仓时使用 `python .\Tools-工具\update_ths_stocks.py --only holdings`。
- 脚本只读本机同花顺缓存、名称表和“我的板块”文件，不读取券商真实交易记录。
- 用户要求把股票信息整理成 Markdown 表格时，优先更新 `10-股票投资/02-股票研究流/我的股票信息.md`，作为 `我的持仓` 和 `我的自选` 的汇总查看页。

## 编译层判断

| 输入 | 输出 |
|---|---|
| 单条来源 | raw 来源笔记 |
| 一组来源入口 | 索引页 |
| 3 个以上同主题来源 | 主题页 |
| 反复出现的判断 | 问题页 |
| 影响行动的选择 | 决策页 |
| 整理过程和异常 | 整理日志 |

## 展开正文的条件

第一轮只读文件名、frontmatter、URL、标题、大小和修改时间。只有以下情况读正文：

- `tags` 为空、无效或无法映射。
- 标签与标题/内容疑似冲突。
- 涉及股票、财务、健康、法律、账号、身份、保险或隐私。
- 需要提取长期知识，而不是只做 raw 归档。

## 整理报告最小结构

| 模块 | 内容 |
|---|---|
| 本次范围 | 扫描目录、文件数量 |
| 分类审计表 | 原文件、原始标签、分类、冲突说明 |
| 处理追踪表 | 原文件、raw 副本、wiki 页面、状态 |
| 处理结果 | raw 数量、更新页面、跳过文件 |
| 风险提示 | 高风险、不确定、敏感内容 |
| 待确认事项 | 需要用户判断的分类、合并、迁移或删除 |
| 日期信息 | 整理日期必填；来源发布日期、收藏日期或文件日期能确认就写，补不上写“来源日期未知” |

状态字段固定为：未处理 / 已复制 raw / 已沉淀 wiki / 已跳过。
