# TradingAgents 多智能体投研架构

本页基于 Simon林的抖音视频转写稿整理。内容是开源项目架构拆解，不构成投资建议。涉及金融和投研判断的部分，只能作为 Agent 设计案例，不能当作交易依据。

## 一句话结论

TradingAgents 的核心价值不是“AI 自动炒股”，而是展示了如何用多个 Agent 模拟一个投研组织：分工收集信息、形成多空观点、互相辩论、由裁判和风险委员会评估，最后由总经理式角色拍板。

## 五层流程

| 层级 | 角色 | 功能 |
|---|---|---|
| 信息采集 | 4 个分析师 | 技术分析、新闻/舆情、基本面、宏观或其他数据源，通过 API 拉取数据 |
| 多空辩论 | 多头 / 空头 | 基于分析报告分别给出看多和看空理由 |
| 裁判判定 | Judge | 阅读多空辩论记录，判断哪一方更有说服力 |
| 风险委员会 | 激进 / 保守 / 中性 | 从风险偏好角度再次争论 |
| 最终拍板 | Boss / Manager | 结合前面所有信息和历史记忆输出最终结论 |

## 设计亮点

- 通过多角色对抗减少单 Agent “输入数据、直接出结论”的黑箱感。
- 把分析、辩论、裁判、风控、拍板拆开，让每个角色只承担一类职责。
- 使用双模型思路：前面的信息收集和初步讨论可用便宜模型，最后拍板角色使用更强模型。
- 引入记忆和反思：不是每次都从零开始，而是让系统根据历史结果修正。
- 通过 API 拉真实数据，避免只靠模型记忆做判断。

## 风险与边界

- 视频转写中项目名被识别为 `Tradeen Agents`，结合标题判断应为 `TradingAgents`，需核验。
- 视频中的示例分析对象包含 Amazon 等公司，具体财务数字和判断均需核验。
- 多 Agent 辩论能提升可解释性，但不能保证结论正确。
- 金融数据、新闻、基本面、技术指标的源质量会直接影响最终输出。
- 这类系统更适合作为“研究辅助”和“框架演示”，不应直接自动交易。
- 视频作者也指出一个关键缺陷：如果四个前置分析师使用便宜模型且抓取数据质量不高，后续辩论、裁判、风控和拍板都可能建立在错误信息上。
